CatchAll-Bird - Important Info

Dear CatchAll-Bird user. Thank you for using this AddOn.

Please note this important limitation regarding the automatic identity creation.

Short Version

Due to a limitation of the Thunderbird AddOn interface, the outgoing mail server (SMTP) of the catch-all account needs to be the default outgoing mail server for the whole installation.
This default server can be changed in the settings (see section "What to do?").
Further, because of this it is only possible to have a single catch-all address enabled when using the automatic identity creation feature.
This is because when sending using an identity created by this AddOn the (at time of sending) default outgoing mail server of the Thunderbird installation is used. With the currently available AddOn interfaces this cannot be changed. There is an open request in the thunderbird bugtracker.

What happens?

Due to a missing feature in the Thunderbird AddOn interface, the outgoing mail server (SMTP) might not be set as expected for identites created by this AddOn.

A mail account is each account you setup in Thunderbird (e.g. you@yourdomain.com or you@yourdomain2.net or you@publicmailprovider.com).

By design each new identity for a prefix is a copy of the first (main) identity of the mail account (except for the new prefix email).
This way the identity settings you made for this mail account should be preserved.

However, there is no way of specifying the per-identity outgoing mail server (SMTP) for an new identity.
And instead of using the known outgoing server of the mail account, Thunderbird uses the default outgoing mail server of the whole installation (at time of sending).

This is an issue if the default outgoing mail server is not the one for your catch-all email account.
If this is not the case Thunderbird will try to send your responses to prefix emails via the wrong outgoing mail server.

Who is affected?

This issue affects users that do not have their catch-all address as default outgoing mail server (SMTP) in the Thunderbird settings or have multiple catch-all addresses.
This is most likely the case if another mail account is present before the catch-all mail account.

You can check the current default outgoing mail server at: Tools -> Account Settings -> Outgoing Server (SMTP)

What to do?

Until Thunderbird adds the required interface (or changes the default behaviour) the options are limited. Due to the missing interface this step needs to be manual.

Change the default SMTP server at
Account-Settings -> Outgoing Server (SMTP) -> Your catch-all account SMTP Server -> Set Default
Further, this usually means that only a single catch-all address can be used. The login credentials are stored in the SMTP server config. Therefore, the SMTP server is either the wrong one or would need to know which domain to sent from.

If this does not work for you please disable the AddOn for this account and go to Account-Settings -> Your Mail Account -> Manage Identities and delete the identities created by the AddOn.
Otherwise, when using these identities the current default outgoing mail server will be used as described above.
Alternatively, manually change the SMTP server inside each of the identities.

Sorry for the inconvenience.
I hope this explanation has been clear. For further questions, details and discussion please go to the GitHub.
